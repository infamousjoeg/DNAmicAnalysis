SSH Keys

SSH Keys are commonly used within large Unix environments to provide access to the estate for administrators. Unfortunately there is no way to centralize visibility and control of these keys, and as a result many organizations simply overlook them when building secure methods of privileged access. Much like passwords, SSH Keys are sued to protect access to privileged user accounts and to verify trust between machine to machine communication protocols, and as such are quite literally the keys to enormous amounts of important data. As such, organizations should proactively secure, manage and monitor the use of privileged SSH keys, just as they should privileged passwords.

SSH Private Keys

By default SSH keys do not expire, and are valid indefinitely. It is not uncommon to see keys that are decades old but are still in use. The longer lived a key is, the more likely it is to have access to a large proportion of the Unix environment as the administrator who owns the key pair gains access to more and more devices. As the public keys are largely anonymous it is difficult to know who has access to which servers and accounts a particular user has access to, which provides a large attack surface for an intruder to potentially misuse.

As with passwords, regular rotation of SSH key pairs is an effective way to reduce the impact of the theft of any key file, particular the private part of the pair. Rotating these credentials reduces the window of opportunity that an attacker has to exploit stolen key before the rotation process invalidates the old credential and replaces it with a new key pair.
