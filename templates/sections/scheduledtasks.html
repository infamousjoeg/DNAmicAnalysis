Scheduled Tasks

As with Windows Services, credentials used by Windows Scheduled Tasks are also stored locally to enable ongoing authentication. Credentials for Scheduled Tasks are stored in the Windows Credential Manager service which, like LSA Secrets, can be accessed by anyone with administrative privileges. Many tools exist that allow for the recovery of passwords used by Scheduled Tasks, and these can readily be exploited by an attacker to discover credentials that could be leveraged as part of the attack chain.

Applying the least-privilege model to Scheduled Tasks is again important. In particular, the he 'allow server operators to schedule tasks' right should be tightly controlled to prevent unnecessary proliferation of credentials for Scheduled Tasks. Where Scheduled Tasks must be implemented, least-privilege should be applied to both the administrators of the system upon which the Task resides as well as to the account used to run the Task itself. Finally, credential rotation again has a role to play in minimizing the impact that stolen Scheduled Task credentials pose.
