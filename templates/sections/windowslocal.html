Windows Server Operating System versions

As the Windows Operating System has evolved over the years, new security features have been added that have decreased the vulnerability of the platform. However due to a variety of factors organizations rarely update their entire environment to the latest available version in a timely manner. It is important to understand the security limitations of older versions of Windows, particularly where those versions are no longer supported by Microsoft themselves, such as Server 2000 or 2003. On these platforms there are far fewer security features, and the risk of credential theft from these devices is much higher than from more modern Windows 2012 or 2016 servers.

Windows Local Administrator accounts

Built-In Administrator Accounts

In Windows environments local Administrator accounts across both desktops and servers are often neglected. With Active Directory offering unified authentication capability across the organization, most IT Admins rely on their domain credentials to authenticate to devices. Local Admin accounts can therefore go unused for long periods of time, with the passwords remaining fixed for months or years.

In some cases organizations also use the same local administrator password across most or even all of their Windows machines. This is sometimes done for simplicity’s sake, and sometimes due to the way that Windows machines are deployed using a template image. In either case it makes an attacker’s job much simpler – all they have to do is compromise one credential to get access to the majority of the Windows estate. Some organizations will rename the account in an attempt to provide some security, but this is of doubtful utility because the "SID 500" account is easily identifiable regardless of its username. Effective management of these passwords is therefore a critical part of any privilege project.

Users With Local Admin Rights on Servers

In most organizations with Active Directory, domain or server administrator accounts are most commonly used for server administration. It is also possible to assign administrative rights to domain accounts directly on servers by adding them to the local Administrator group.

When administrative privileges are assigned in this manner, centralized management and monitoring becomes extremely difficult. It is much more difficult to revoke administrative rights, such as when a user changes roles or leaves a business entirely.

What’s worse is that these users, if compromised, could allow an attacker to escalate privilege. Local administrator accounts can be used to extract credentials and credential hashes for other accounts that have used the same server, such as a domain administrator account or a highly-privileged service account. Attackers can install keystroke loggers to look for any typed passwords or run tools like Mimikatz to obtain privileged account hashes.
