Hard-coded Web Server Credentials

Windows IIS Applications

It is very common for application servers to require credentials for downstream systems in order to store or retrieve data, typically from a database server. IIS-hosted applications are very likely to have connection strings that contain credentials for SQL database servers. By default, these connection strings are located in the web applicationâ€™s web.config file as a plain-text XML attribute.

Leaving hard-coded credentials for powerful accounts in config files gives attackers easy access to sensitive information. Any compromise of the server is likely to lead to the attacker finding these hard-coded credentials and exploiting them to gain access to the downstream system. Additionally it is likely this credential will not be changed frequently due to the potential downtime ramifications as well as the difficulty of updating the configuration of a distributed, multi-node web applications.
