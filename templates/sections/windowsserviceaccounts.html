Windows Services

The Windows Local Security Authority (LSA) is responsible for managing local security policy, including authenticating services. This service uses a sub-system called LSA Secrets to store encrypted credentials in the SECURITY hive of the Windows registry, which in theory protects these credentials from unauthorized access. However, the encryption keys for these passwords are also stored in LSA Secrets and can be accessed by anyone with administrative privileges on the Windows machine. With these keys, the user can then simply decrypt the credential and retrieve the plain-text password. The password for any Windows service is therefore easily accessible to anyone who obtains administrative privileges on a Windows machine.

A properly designed least-privilege model is the best defense against theft of Service Account passwords. Restricting administrative privileges on Windows machines reduces the risk of these passwords being stolen in the first place, whilst a similar restriction placed on the Service Accounts themselves means that if their credentials are stolen, the potential impact is limited. No Service Account should ever be granted Domain Admin privileges, for example. Also, credential rotation of Service Accounts, whilst difficult to implement, ensures that attackers have a limited window of opportunity to exploit any stolen credentials before the password is changed.
