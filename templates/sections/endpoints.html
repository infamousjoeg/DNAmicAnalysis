Privileged Accounts on Endpoints

Business user accounts with local administrator rights make it much easier for an attacker to gain a foothold in the network and steal even more powerful credentials. Direct administrator rights on workstations, for example, means that attackers can install keystroke loggers to look for any typed passwords or run tools like mimikatz to get privileged hashes. In addition, the attack surface is unnecessarily widened because each of these business user accounts, which should be non-privileged, has now become a privileged ID that can be exploited.

Windows Desktop Operating System versions

As with Windows Server, the Windows desktop operating system has evolved over the years, adding security improvements with each new version. Just as with servers, organizations rarely upgrade their entire desktop estates to the latest versions available, potentially leaving behind older machines that can be exploited more readily than the latest Windows 10 edition. It is therefore important to understand the full breadth of desktop machines on the network with a view to mitigating any weaknesses in older machines, especially those with unsupported versions of Windows such as Windows XP.

Local Admin accounts on desktops

As with servers, built-in local administrator accounts on desktops are often a neglected risk. With hundreds or thousands of desktops to manage, IT teams rely on their privileged domain credentials to access devices and make changes or repairs leaving built-in accounts to be forgotten. What's worse is that managing desktop local administrator passwords is even more difficult than those of servers, because many of these devices are mobile and are not guaranteed to be attached to the network at any point in time, posing a formidable challenge to automated password management solutions.

Endpoints are not usually the end-goal for a determined, advanced threat actor but they are often the starting point, and poorly managed local administrator accounts - especially those where passwords are common across devices - provide a convenient and easy to exploit path to discover additional privileged accounts that can then be targeted for compromise, leading to access into more secure areas of the network. The first line of password security should therefore start on the desktop.

Users with Local Admin rights on desktops

Endpoints such as Windows desktops are often the first point-of-entry for attackers seeking to penetrate an organization. Desktops and laptops are the most exposed parts of the network, being operated by users who are not particularly IT-security conscious and who are the most likely to fall victim to typical exploitation tactics – phishing, malware traps, phone scams and the like. All it takes is one user to make the wrong decision, and attackers can gain access to machines.

However, endpoints are not usually the end-goal for a determined, advanced threat actor. Sophisticated attackers will use a compromised endpoint as the starting point for a deeper attack that targets critical assets such as database servers which are much more likely to contain valuable data. To obtain access to this information usually requires higher than standard user privileges.

UsersWithLocalAdminDesktops users have Local Admin rights on desktops (not via Domain groups)

The privileges of these users are difficult to manage, as they are set directly on the device in question and not centrally via Active Directory

Excessive provisioning of Local Admin access to users on desktops can lead to compromise in several ways:

  • Local Admins have the rights to install software – including malware
  • They can potentially remove software that 'gets in the way' – anti-virus, for example
  • Local Admins have access to stores of other user authentication data – such as LSASS, or cached Service Account passwords
