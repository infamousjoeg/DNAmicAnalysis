Root Accounts

Securing root accounts is one of the fundamental steps that IT administrators must take to protect their Linux and Unix infrastructure. Various recommendations and best practice guides for securing root accounts have been published, including disabling these accounts or preventing remote access using them, but they nevertheless remain on every Unix device with the highest privileges assigned to them. They must be managed properly to prevent an external attacker or even an insider with malicious intent misusing them.

Static root accounts with shared passwords or SSH Keys offer an enticing path of least resistance for attackers. Rather than wasting time looking at non-root accounts, attackers will start by looking for weaknesses in root access because it is guaranteed to have the privileges needed to be successful. If root accounts never change and/or share a common credential (for example to facilitate the work of the support team), an attacker has everything they need to move laterally and circumvent traditional command restrictions such as sudo.
